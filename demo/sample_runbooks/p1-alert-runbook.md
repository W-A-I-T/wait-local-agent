# P1 Service Desk Alert Runbook

Use this runbook for high-impact client alerts.

1. Confirm the alert affects more than one user or a business-critical workflow.
2. Capture client, site, affected system, start time, and observed impact.
3. Create an internal note with facts only. Avoid speculation.
4. Draft a client-safe update with current impact, next action, and next update time.
5. Assign an owner and keep the approval queue updated before posting any PSA note.

Client-safe update pattern:

```text
We are investigating an issue affecting <system>. Current impact is <impact>. The next action is <action>, and the next update will be sent by <time>.
```
