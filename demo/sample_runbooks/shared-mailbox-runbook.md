# Shared Mailbox Access Runbook

Use this runbook when a client requests access to a shared mailbox.

1. Confirm the requester is authorized by the client contact list.
2. Grant only mailbox-level access required for the role.
3. Do not grant tenant administrator rights for a mailbox request.
4. Record the mailbox, requester, approver, and permission level in the ticket.
5. Send a client-safe response that confirms the change and reminds the requester to restart Outlook or refresh web mail.

Suggested internal note:

```text
Validated shared mailbox access request. Permission scope is mailbox-only; no tenant administrator rights required.
```
